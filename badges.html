          While working on my defunct blog's makeover, I needed some interesting sidebar fillers/badges. Before the common Twitter feed, I believed that some GitHub nifty feeds would be a better fit to start with.

          What kind of feeds?

          Lists of my public, forked or watched repositories. Or maybe a feed of the latest commits on a project's dedicated page.

          Why build something from scratch?

          After a quick Google search, I came to the conclusion that no available library/script would do the job. TBH, I haven't tried any - code speaks for itself.

          Constants, hardcoded options and serving a single API per script were enough reasons to get me coding.

          Current (limited) feature set

          • APIs: commits, forked, repos, watched
          • Basic result filtering
          • Customizable templates
          • Maximum limit (with footer link to show more)
          • Gravatar
          • more...

          Future plans

          As you might have noticed, while the feature set is varied and quite satisfactory for all general purposes, I am still planning on adding support for:

          • tests
          • callbacks
          • a smarter templating engine
          • more of GitHub's APIs
          • result sorting/ordering
          • and the unavoidable bug fixes

          Credits / Inspiration

          Like I said above, other badges exist out there. From the minute I searched and until now, here are the ones I stumbled on and which might have inspired me:

          • DrNic's GitHub Badge, for a list of your public repos.
          • mattn's jQuery GitHub Badge, for a list of your latest commits.
          • kentbrew's GitHub Social Badge, visualize projects and followers.
          • heipei's GitHub Commit Badge, for displaying the latest (or a specific commit) for a github-project.
          Remember, I haven't tried any. I suspect some of them aren't even maintained and still using GitHub's v1 API.

          For Gravatar support and pretty date formats:

          • MD5 (by Masanao Izumo), copied from mattn's script
          • John Resig's prettyDate()
